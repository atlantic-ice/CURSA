# VKR 100% Fast-Track Checklist

Цель: довести проект до состояния, когда старт написания ВКР полностью закрыт по фактам, а не по декларациям.

## Definition of Done (VKR-Start 100%)

- Единый официальный статус готовности зафиксирован и не противоречит другим документам.
- Проверки окружения и тестов запускаются одной командой на Windows.
- Есть воспроизводимый отчет по структурным метрикам, тестам и демо-готовности.
- Подготовлен комплект артефактов для глав 1-4 ВКР.
- Подготовлен сценарий демонстрации и доказательства повторяемости результатов.

## Быстрый запуск проверки

```powershell
pwsh -File scripts/vkr_fast_track.ps1
```

Если используется Windows PowerShell 5.1:

```powershell
powershell -ExecutionPolicy Bypass -File scripts/vkr_fast_track.ps1
```

Отчеты появятся в каталоге:

- .artifacts/vkr_ready/<timestamp>/VKR_FAST_TRACK_REPORT.md
- .artifacts/vkr_ready/<timestamp>/vkr_fast_track_report.json

## План закрытия на максимальной скорости

1. Прогнать автоматический аудит и получить факт-отчет.
2. Закрыть все FAIL/MISSING из отчета без изменения scope ВКР.
3. Обновить единый статусный документ по фактическим результатам.
4. Провести повторный прогон до статуса VKR_READY_100.
5. Зафиксировать результаты в комплекте материалов ВКР.

## Минимальный пакет артефактов для старта написания

- Официальный статус проекта на текущую дату.
- Отчет автоматического fast-track аудита.
- Отчет по тестам backend.
- Набор демонстрационных кейсов документов.
- Скриншоты/выгрузки ключевых экранов и API-ответов.

## Принцип качества

Любой пункт считается закрытым только после автоматической или воспроизводимой ручной проверки с сохранением результата в репозитории.

## Продолжение работы над ВКР (оперативный трек)

1. Вычитать и выровнять текст глав 1-4 под стиль кафедры: терминология, формулировки цели, задач и новизны.
2. Сверить все численные показатели в тексте с фактическими отчетами fast-track и тестов.
3. Дособрать демонстрационный пакет: скриншоты интерфейса, 2-3 показательных API-ответа, краткий сценарий показа.
4. Подготовить доклад на 7-10 минут: проблема, архитектура, результаты, практическая ценность.
5. Подготовить ответы на типовые вопросы комиссии: почему выбран этот стек, как подтверждена точность, как обеспечена масштабируемость.

## Контрольные точки перед защитой

- Текст ВКР не содержит неподтвержденных цифр и заявлений.
- Все ссылки на артефакты воспроизводимы локально в репозитории.
- Демонстрационный сценарий выполняется за один проход без ручных правок.
- Есть финальный прогон fast-track в день сдачи материалов.

## Runbook на день защиты (T-60 .. T-0)

### T-60 минут: контроль окружения

1. Открыть корень проекта `CURSA`.
2. Запустить проверку:

```powershell
powershell -ExecutionPolicy Bypass -File scripts/vkr_fast_track.ps1
```

3. Убедиться, что в отчете итоговый статус: `VKR_READY_100`.

### T-45 минут: контроль артефактов

Проверить наличие файлов:

- `VKR_NIKITA_CURSA_2026.docx`
- `VKR_MAIN.md`
- `VKR_COMPLETION_REPORT.md`
- `VKR_FINAL_CHECKLIST.md`
- `VKR_SUBMISSION_SUMMARY.md`
- `.artifacts/vkr_ready/<latest>/VKR_FAST_TRACK_REPORT.md`

### T-30 минут: прогон демо-сценария

1. Запустить backend и frontend.
2. Проверить базовый пользовательский путь:
	- загрузка документа,
	- запуск анализа,
	- просмотр отчета,
	- скачивание результата.
3. Проверить `/api/health`.

### T-15 минут: контроль доклада

Фиксированный каркас доклада 7-10 минут:

1. Проблема и актуальность.
2. Архитектура и 15 валидаторов.
3. Подтвержденные результаты (VKR_READY_100, 9/9 PASS).
4. Практическая ценность и масштабируемость.

### T-0: финальная фиксация

- Сохранить путь к последнему fast-track отчету.
- Использовать только подтвержденные числа из официального статуса и последнего отчета.

## Мини-шпаргалка ответов комиссии

- Почему этот стек: Python удобен для DOCX и валидации, React/TypeScript дает управляемый интерфейс и типобезопасность.
- Как подтверждена готовность: автоматическим fast-track аудитом со статусом `VKR_READY_100`.
- Почему результат воспроизводим: проверка запускается одной командой и формирует артефакты в `.artifacts/vkr_ready/`.
- Что с масштабируемостью: архитектура сервисная, обработка документов проходит в предсказуемом режиме.
