# 🤖 Быстрый старт с автономным AI-агентом

## ✅ Готово к работе

Ваш проект настроен для автономной разработки с AI-агентом. Агент работает как senior-разработчик и **не требует микроменеджмента**.

## 🚀 Как использовать

### Просто ставьте задачи:

```
Добавь валидацию email в форму регистрации
```

```
Оптимизируй загрузку документов
```

```
Исправь баг с кодировкой в парсере
```

Агент сам:
- ✅ Реализует функциональность
- ✅ Напишет тесты
- ✅ Добавит обработку ошибок
- ✅ Задокументирует код
- ✅ Проверит что всё работает
- ✅ Предложит следующие шаги

### НЕ нужно:

❌ Спрашивать "добавить ли тест?"
❌ Уточнять "какой подход использовать?"
❌ Подтверждать исправление багов
❌ Выбирать между вариантами реализации

## 📋 Примеры задач

### Простые задачи
```
Добавь логирование в функцию process_document
```

Агент добавит правильное логирование без вопросов.

### Средние задачи
```
Создай API endpoint для получения списка профилей
```

Агент создаст:
- Endpoint с правильной типизацией
- Обработку ошибок
- Unit и integration тесты
- Документацию

### Сложные задачи
```
Реализуй систему кэширования для документов
```

Агент:
- Выберет подходящую технологию (Redis/in-memory)
- Реализует кэш с TTL
- Добавит инвалидацию
- Напишет тесты
- Оптимизирует производительность
- Предложит дальнейшие улучшения

## 🎯 Когда агент спросит

Только в критических случаях:

```
⚠️  Необходим API ключ для внешнего сервиса
⚠️  Неясно какая бизнес-логика нужна для расчёта скидок
⚠️  Требуется полная переработка архитектуры базы данных
```

## 📁 Что настроено

- `.github/copilot-instructions.md` - основные правила
- `.github/instructions/*.instructions.md` - специализированные инструкции
- `.vscode/settings.json` - оптимальные настройки редактора
- `.vscode/extensions.json` - рекомендуемые расширения

## 🔧 Первый запуск

1. Откройте проект в VS Code Insiders
2. Установите рекомендуемые расширения (VS Code предложит)
3. Убедитесь что GitHub Copilot активен
4. Начинайте ставить задачи в чате!

## 💡 Советы

**Хорошая формулировка задачи:**
```
Добавь проверку формата ГОСТ для списка литературы
```

**Плохая формулировка:**
```
Может быть стоит как-то проверять список литературы?
```

**Агент лучше работает с:**
- Конкретными задачами
- Глаголами действия (добавь, исправь, оптимизируй)
- Указанием контекста если нужно

## 🎨 Стиль работы агента

После выполнения задачи агент сообщит:

```
✅ Добавлена валидация email

Реализовано:
- Функция validate_email с regex проверкой
- Обработка edge cases (пустая строка, None)
- 8 unit тестов
- Docstring с примерами

Далее: добавить валидацию телефона, настроить логирование ошибок валидации
```

Вы можете:
- Согласиться с предложением: "Да, добавь"
- Выбрать что-то одно: "Только логирование"
- Поставить новую задачу: "Добавь экспорт в Excel"
