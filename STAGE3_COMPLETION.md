## 🆕 Новые валидаторы

### ImageValidator (Правило 20)

**Назначение**: Проверяет оформление рисунков и диаграмм

**Проверяет:**

- ✓ Наличие подписей для всех изображений
- ✓ Правильность формата подписей (Рисунок X - Описание)
- ✓ Последовательность нумерации рисунков (1, 2, 3...)
- ✓ Наличие ссылок на рисунки в основном тексте
- ✓ Наличие "Список рисунков" при >1 рисунке

**Интеграция:**

- Файл: `backend/app/services/validators/image_validator.py` (340 строк)
- Документация: `IMAGE_VALIDATOR.md`
- Статус: 100% функционален

**Пример проверки:**

```
Документ содержит 3 рисунка:
✓ Рисунок 1 - Архитектура системы (найдена подпись)
✓ Рисунок 2 - Модули компонентов (найдена подпись)
✓ Рисунок 3 - Диаграмма взаимодействия (найдена подпись)
✓ Все рисунки упоминаются в тексте
✓ Присутствует "Список рисунков"
```

### AppendixValidator (Правила 22-23)

**Назначение**: Проверяет оформление приложений документа

**Проверяет:**

- ✓ Правильность нумерации приложений (ПРИЛОЖЕНИЕ А, Б, В или 1, 2, 3)
- ✓ Консистентность стиля нумерации (не смешивание букв и цифр)
- ✓ Наличие названия каждого приложения
- ✓ Форматирование заголовков (ПРОПИСНЫЕ буквы)
- ✓ Наличие ссылок на приложения в основном тексте

**Интеграция:**

- Файл: `backend/app/services/validators/appendix_validator.py` (380 строк)
- Документация: `APPENDIX_VALIDATOR.md`
- Статус: 100% функционален

**Пример проверки:**

```
Документ содержит 3 приложения:
✓ Приложение А - Спецификация API (правильный формат)
✓ Приложение Б - Исходный код (правильный формат)
✓ Приложение В - Расширенные таблицы (правильный формат)
✓ Нумерация последовательна (А → Б → В)
✓ Все приложения упоминаются в основном тексте
```
