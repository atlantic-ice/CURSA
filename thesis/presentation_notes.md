## СЛАЙД 2: Актуальность (1 мин)

**Почему эта тема актуальна?**

Каждый студент при написании курсовых и дипломных работ сталкивается с необходимостью оформления документов строго по ГОСТ. Это занимает огромное количество времени.

**Проблемы существующего подхода:**
- Ручная проверка 50-страничного документа занимает от 45 минут до 1.5 часов
- Человеческий фактор приводит к пропуску 8-12% ошибок
- Разные проверяющие дают непоследовательные оценки

**Решение** — программный комплекс CURSA, который автоматизирует этот процесс.

## СЛАЙД 3: Цель и задачи (1 мин)

**Цель работы** — разработать программный комплекс для автоматизации проверки и оформления текстовых документов.

Для достижения цели были поставлены следующие **задачи**:

3. Спроектировать архитектуру системы
4. Разработать алгоритмы проверки документов
5. Реализовать программный комплекс
6. Провести тестирование

---

## СЛАЙД 4: Архитектура (1.5 мин)

**Архитектура системы** — клиент-серверная модель.

**Frontend** — React-приложение с TypeScript:
- Страница загрузки документов
- Страница просмотра отчётов
- История проверок
- Управление профилями

**Backend** — Flask API на Python:
- Подсистема валидации с 10 валидаторами
- Подсистема автокоррекции
- REST API для взаимодействия с клиентом

**База данных** — PostgreSQL для пользователей и результатов, JSON-файлы для профилей требований.

---

## СЛАЙД 5: Валидаторы (2 мин)

**Ключевая часть системы** — 10 валидаторов, покрывающих 19 правил ГОСТ 7.32-2017:

| Валидатор | Что проверяет |
|-----------|---------------|
| FontValidator | Шрифт Times New Roman, размер 14pt |
| MarginValidator | Поля страницы 3x1.5x2x2 см |
| ParagraphValidator | Межстрочный интервал 1.5, отступ 1.25 см |
| HeadingValidator | Заголовки разделов |
| TableValidator | Оформление таблиц |
| FormulaValidator | Нумерация формул |
| ImageValidator | Рисунки и подписи |
| BibliographyValidator | Список литературы |
| AppendixValidator | Приложения |
| StructureValidator | Общая структура документа |

Каждый валидатор реализован как независимый модуль — это обеспечивает расширяемость системы.

---

## СЛАЙД 6: Алгоритм работы (1 мин)

**Как работает система:**

1. Пользователь загружает DOCX файл через веб-интерфейс
2. Backend сохраняет файл и запускает Validation Engine
3. Validation Engine последовательно запускает 10 валидаторов
4. Каждый валидатор проверяет свой аспект документа
5. Результаты агрегируются в единый отчёт
6. Пользователь получает:
   - Список ошибок с позициями в документе
   - Оценку соответствия (0-100%)
   - Рекомендации по исправлению
   - Возможность автокоррекции
