# ЗАКЛЮЧЕНИЕ

В результате выполнения выпускной квалификационной работы был разработан программный комплекс CURSA для автоматизации проверки и оформления текстовых документов в соответствии с требованиями нормоконтроля.

В ходе выполнения работы были решены следующие задачи:

1. **Проведён анализ предметной области**, включающий изучение требований ГОСТ 7.32-2017, ГОСТ Р 7.0.5-2008, ГОСТ 7.1-2003 и типовых требований учебных заведений к оформлению документов. Выявлены основные категории правил нормоконтроля: требования к шрифтовому оформлению, параметрам страницы, тексту и структуре документа. Определены типовые ошибки, допускаемые при оформлении документов.

2. **Выполнен обзор существующих аналогов** — программных средств проверки документов. Проведён сравнительный анализ инструментов Microsoft Word, «Справка-М» и Antijob. Определены преимущества и недостатки каждого решения. Сделан вывод об отсутствии на рынке комплексных решений, обеспечивающих полный цикл проверки с возможностью автокоррекции и настраиваемыми профилями.

3. **Спроектирована архитектура программного комплекса** на основе клиент-серверной модели с разделением на frontend (React) и backend (Flask). Определена структура компонентов и взаимодействие между ними. Спроектирована база данных с пятью основными таблицами. Разработано REST API с эндпоинтами для аутентификации, работы с документами и управления профилями.

4. **Разработаны алгоритмы проверки документа** на основе модульной системы валидаторов. Реализовано 10 валидаторов, обеспечивающих проверку 19 требований ГОСТ 7.32-2017: StructureValidator, FontValidator, MarginValidator, ParagraphValidator, HeadingValidator, BibliographyValidator, TableValidator, FormulaValidator, ImageValidator, AppendixValidator.

5. **Реализован программный комплекс** с функциями загрузки документов через веб-интерфейс, проверки на соответствие требованиям нормоконтроля, формирования детализированных отчётов об ошибках и автоматического исправления ошибок форматирования. Серверная часть реализована на Python с использованием Flask, клиентская — на React с TypeScript.

6. **Проведено тестирование** программного комплекса на нескольких уровнях: unit-тесты (89 тестов), интеграционные тесты (32 теста), функциональные тесты (18 тестов). Общее покрытие кода тестами составило 70%.

Разработанный программный комплекс обеспечивает:

- **Сокращение времени проверки** документа объёмом 50 страниц до 6.6 секунд, что в 400-800 раз быстрее ручной проверки (45-90 минут)
- **Точность выявления ошибок** 96.6% (среднее F1-score для всех валидаторов)
- **Автоматическое исправление** 80% обнаруженных ошибок форматирования
- **Поддержку настраиваемых профилей** требований для различных учебных заведений
- **Веб-интерфейс**, работающий без установки дополнительного программного обеспечения

Научная новизна работы заключается в разработке модульной архитектуры системы валидации документов с поддержкой настраиваемых профилей требований для различных учебных заведений и в реализации многопроходного алгоритма автоматического исправления ошибок форматирования с сохранением исходного содержания документа.

Практическая значимость работы состоит в создании работающего программного продукта, который может использоваться студентами и преподавателями для автоматизации процесса проверки документов, сокращения временных затрат на подготовку письменных работ и повышения качества их оформления.

## Перспективы развития

Разработанный программный комплекс имеет перспективы дальнейшего развития в следующих направлениях:

1. **Расширение списка поддерживаемых форматов** — добавление поддержки форматов DOC, ODT, PDF
2. **Улучшение подсистемы автокоррекции** — увеличение доли автоматически исправляемых ошибок
3. **Разработка мобильного приложения** — для работы с системой на мобильных устройствах
4. **Интеграция с системами управления обучением** (Moodle, Google Classroom)
5. **Добавление функций проверки уникальности текста** (антиплагиат)
6. **Корпоративная версия** — с поддержкой многопользовательского режима, расширенной аналитикой и администрированием
