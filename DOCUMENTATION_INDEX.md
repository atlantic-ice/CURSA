```
1. What did I complete yesterday?
2. What am I doing today? (from QUICK_REFERENCE)
```

### End of Day

```
1. All tests passing? (run pytest)
2. Code committed? (git push)
3. Code review requested? (GitHub PR)
4. Any blockers for tomorrow?
5. Update PRODUCTION_READINESS_DASHBOARD (1 min)
```

### Friday (3 hours)

```
1. Code review all PRs for the week
2. Merge to main
3. Run full test suite

5. 15:00 Team standup (30 min)
   - What accomplished this week?
   - Any risks for next week?
   - Adjusted estimates?
   - Dependencies on other teams?
6. Plan next week (30 min)
```

## 📞 ESCALATION & HELP

### If you're blocked:

1. **Post immediately:** #cursa-production Slack
2. **Mention:** @tech-lead with problem description
3. **Create ticket:** GitHub issue with "BLOCKED" label
4. **Resolution:** Tech lead responds within 4 hours for P1 blockers

### Common blockers & quick fixes:

- Redis not running? → `docker run -d -p 6379:6379 redis:latest`
- SendGrid key wrong? → Check .env, update from dashboard
- OAuth not working? → Verify callback URL in app settings
- Tests failing? → Run single test, check fixtures
- Need code review? → Tag in PR, @tech-lead will review same day

## 🎓 ASSUMPTIONS & DEPENDENCIES

### External Dependencies

- ✅ SendGrid account (for emails)
- ✅ Google OAuth credentials
- ✅ GitHub OAuth credentials
- ✅ Stripe test account (Week 2)
- ✅ Yookassa test account (Week 2)
- ✅ Redis server
- ✅ PostgreSQL database

### Assumptions

- Docker installed locally
- Python 3.8+ available
- Node.js 14+ available
- Git for version control
- Team knows Flask + React

### Risks

- 3rd party API outages (OAuth, Stripe, SendGrid)
  → Mitigate: Setup fallback options
- Scope creep (gold-plating features)
  → Mitigate: Strict MVP definition, feature freeze at Week 17
- Team unavailability
  → Mitigate: Cross-training, documentation, backup devs
