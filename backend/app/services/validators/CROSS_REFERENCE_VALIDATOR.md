# CrossReferenceValidator Документация

## 📋 Описание

**CrossReferenceValidator** проверяет правильность перекрестных ссылок, нумерации разделов и ссылок на объекты в документе согласно ГОСТ 7.32-2017.

Валидатор проверяет:

- ✅ Иерархию нумерации разделов (1, 1.1, 1.1.1)
- ✅ Корректность переходов между уровнями нумерации
- ✅ Существование объектов на которые ссылаются
- ✅ Валидность перекрестных ссылок в документе

## 🎯 Правила ГОСТ

### Правило 14: Нумерация разделов

**Требование:** Разделы документа должны пронумерованы в правильной иерархии:

- Первый уровень: 1, 2, 3, ...
- Второй уровень: 1.1, 1.2, 1.3, ..., 2.1, 2.2, ...
- Третий уровень: 1.1.1, 1.1.2, ..., 1.2.1, ...

**Примеры допустимого:**

```
1. Введение
1.1. История вопроса
1.1.1. Предыдущие исследования
1.1.2. Современное состояние
1.2. Постановка задачи
2. Методология
2.1. Выбор метода
2.1.1. Причины выбора
```

**Примеры ошибок:**

```
1. Введение
1.1. История
1.3. Скачок нумерации (должно быть 1.2)
2. Методология
2.0.1. Неправильная иерархия (0 в середине)

1. Введение
1.1.1. Пропущен уровень 1.1
```

### Правило 26: Ссылки на объекты

**Требование:** Все ссылки на таблицы, рисунки и формулы должны указывать на существующие объекты с правильным номером.

**Примеры допустимого:**

```
В таблице 1 приведены...
На рисунке 2 видно...
Из формулы (3) следует...
```

**Примеры ошибок:**

```
В таблице 10 приведены... // Таблицы 10 нет в документе
На рисунке 5 видно... // Рисунок 5 не существует
Из формулы (99) следует... // Формулы (99) нет
```

### Правило 27: Перекрестные ссылки

**Требование:** Ссылки на разделы должны корректно указывать на существующие разделы с правильной нумерацией.

**Примеры допустимого:**

```
Как указано в разделе 2.1, метод выбран...
При рассмотрении раздела 3.2.1 видно...
см. раздел 1.1 для деталей
```

**Примеры ошибок:**

```
Как указано в разделе 5.1 (раздела 5.1 нет)
см. раздел 2.3.1 (нет третьего уровня в разделе 2.3)
По данным раздела 10 (раздела 10 нет)
```

```python
# CrossReferenceValidator включён по умолчанию
```



## 🐛 Типы ошибок

### Правило 14: Нумерация разделов

| Тип                        | Уровень | Описание                             |
| -------------------------- | ------- | ------------------------------------ |
| `numbering_jump`           | ERROR   | Пропуск уровня нумерации (1.1 → 1.3) |
| `hierarchy_violation`      | ERROR   | Нарушение иерархии (1 → 1.0.1)       |
| `duplicate_numbering`      | WARNING | Повторяющаяся нумерация              |
| `invalid_level_transition` | ERROR   | Неправильный переход между уровнями  |

### Правило 26: Ссылки на объекты

| Тип                         | Уровень | Описание                         |
| --------------------------- | ------- | -------------------------------- |
| `invalid_table_reference`   | ERROR   | Ссылка на несуществующую таблицу |
| `invalid_figure_reference`  | ERROR   | Ссылка на несуществующий рисунок |
| `invalid_formula_reference` | ERROR   | Ссылка на несуществующую формулу |
| `malformed_reference`       | WARNING | Неправильный формат ссылки       |

### Правило 27: Перекрестные ссылки

| Тип                         | Уровень | Описание                        |
| --------------------------- | ------- | ------------------------------- |
| `invalid_section_reference` | ERROR   | Ссылка на несуществующий раздел |
| `ambiguous_reference`       | WARNING | Неоднозначная ссылка на раздел  |
| `missing_section_number`    | ERROR   | Раздел без номера               |

## ✅ Рекомендации по исправлению

### Проблема: Пропуск в нумерации разделов

**Диагноз:**

```
Неправильный переход нумерации: 1.1 → 1.3 (пропущено 1.2)
```

**Решение:**

1. Рассчитать правильную нумерацию:
   - Если раздел 1.3 - это третий подраздел под 1, переименовать на 1.2
   - Если раздел 1.3 - это четвёртый подраздел, добавить пропущенный 1.2
2. Обновить все ссылки на этот раздел
3. Зачем добавить обновленый рисунок раздела в оглавлении

### Проблема: Нарушена иерархия нумерации

**Диагноз:**

```
Нарушена иерархия нумерации: после 1 ожидается 1.1, найдено 1.0.1
```

**Решение:**

1. Проверить правильность иерархии Outline (через View → Navigation Pane)
2. Переименовать раздел 1.0.1 на 1.1
3. Убедиться что outline правильно отражает структуру

### Проблема: Ссылка на несуществующий объект

**Диагноз:**

```
Таблица 5 не существует в документе
```

**Решение:**

1. Проверить что таблица 5 действительно есть в документе
2. Если таблицы нету:
   - Изменить номер ссылки на существующий
   - ИЛИ добавить то содержа что описывается в ссылке
3. Если таблица не пронумерована:
   - Добавить подпись в формате "Таблица 5 - Описание"

### Проблема: Неправильная ссылка на раздел

**Диагноз:**

```
Раздел 5.1 не найден в документе
```

**Решение:**

1. Проверить что раздел существует
2. Если раздел есть - обновить номер в ссылке
3. Если раздела нет:
   - Удалить ссылку
   - ИЛИ создать недостающий раздел

## 🔍 Алгоритм работы

### 1. Извлечение разделов (Правило 14)

```
Для каждого параграфа:
  1. Проверить стиль (Heading 1, 2, 3, etc.)
  2. Извлечь номер раздела (регулярное выражение)
  3. Сохранить в список номеров разделов
```

### 2. Проверка иерархии (Правило 14)

```
Для каждого номера раздела:
  1. Разбить на уровни (1.2.3 → [1, 2, 3])
  2. Проверить что каждый уровень чередуется правильно
  3. Убедиться что после N.X нет N.X.Y если N.X ещё не встречалась
```

### 3. Поиск перекрестных ссылок (Правило 27)

```
Для каждого параграфа:
  1. Найти все ссылки на разделы (регулярные выражения)
  2. Извлечь номер раздела из ссылки
  3. Проверить что этот номер есть в списке разделов
```

### 4. Проверка ссылок на объекты (Правило 26)

```
Для каждого параграфа:
  1. Найти ссылки на таблицы (таблица N)
  2. Найти ссылки на рисунки (рисунок N)
  3. Найти ссылки на формулы (формула N)
  4. Проверить что указанные номера существуют
```

## 💡 Примеры использования

### Пример 1: Проверка всех 3 правил

```python
from app.services.validators import CrossReferenceValidator
from docx import Document

doc = Document('thesis.docx')
validator = CrossReferenceValidator()
result = validator.validate(doc, {})

if result.passed:
    print("✅ Перекрестные ссылки корректны")
else:
    print(f"❌ Найдено {len(result.issues)} проблем:")
    for issue in result.issues:
        print(f"  - Правило {issue.rule}: {issue.message}")
```

### Пример 2: Анализ ошибок по типам

```python
result = validator.validate(doc, {})

# Разбить ошибки по типам
errors_by_type = {}
for issue in result.issues:
    if issue.type not in errors_by_type:
        errors_by_type[issue.type] = []
    errors_by_type[issue.type].append(issue)

# Вывести группированным образом
for issue_type, issues in errors_by_type.items():
    print(f"\n{issue_type}: {len(issues)} проблем")
    for issue in issues[:3]:  # Показать первые 3
        print(f"  - {issue.message}")
```

### Пример 3: Извлечение информации о разделах

```python
result = validator.validate(doc, {})

# Получить информацию о найденных разделах
# (эта информация может быть в документации результата)
sections = result.get('sections', [])
print(f"Найдено разделов: {len(sections)}")
for section in sections:
    print(f"  - {section['number']}: {section['title']}")
```

## 🔗 Интеграция с другими валидаторами

**CrossReferenceValidator** работает в связке с:

- **StructureValidator** - базовая проверка структуры документа
- **HeadingValidator** - проверка стиля заголовков
- **TableValidator** - проверка таблиц и нумерации
- **ImageValidator** - проверка рисунков и нумерации
- **FormulaValidator** - проверка формул и нумерации

## 📝 Примечания

1. **Иерархия разделов** - критична для правильной структуры. Используйте встроенные стили Heading 1, 2, 3 в Word.

2. **Нумерация** - должна быть последовательной и без пропусков. Не используйте ручную нумерацию.

3. **Ссылки на разделы** - используйте встроенные Cross-reference (References → Cross-reference) для автоматических ссылок.

4. **Обновление ссылок** - перед финальной проверкой обновите все полученные (Ctrl+A, затем F9 или через Edit → Select All → Update Fields).

5. **Регулярные выражения** - валидатор использует регулярные выражения для поиска ссылок. Убедитесь что ваши ссылки соответствуют стандартным форматам.
