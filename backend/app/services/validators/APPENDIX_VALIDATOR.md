# AppendixValidator - Валидатор приложений (Правила 22-23)

## Описание

`AppendixValidator` проверяет правила оформления приложений в документе согласно ГОСТ 7.32-2017.

**Покрытие**: Правила 22-23 - Приложения

## Проверяемые правила

### 1. ✓ Нумерация приложений

Приложения должны иметь четкую нумерацию. Допускаются два варианта:

**Вариант 1: Буквы (рекомендуется)**

```
ПРИЛОЖЕНИЕ А - Название
ПРИЛОЖЕНИЕ Б - Название
ПРИЛОЖЕНИЕ В - Название
```

**Вариант 2: Цифры**

```
ПРИЛОЖЕНИЕ 1 - Название
ПРИЛОЖЕНИЕ 2 - Название
ПРИЛОЖЕНИЕ 3 - Название
```

**Неправильное:**

```
ПРИЛОЖЕНИЕ А - ...
ПРИЛОЖЕНИЕ В - ...    ❌ Пропущено ПРИЛОЖЕНИЕ Б
```

### 2. ✓ Формат заголовка приложения

Заголовок должен быть в следующем формате:

```
ПРИЛОЖЕНИЕ X - Название приложения
```

Где:

- **ПРИЛОЖЕНИЕ** - напечатано ПРОПИСНЫМИ буквами
- **X** - буква (А, Б, В...) или цифра (1, 2, 3...)
- **Название** - описание содержимого приложения

**Примеры правильного:**

```
ПРИЛОЖЕНИЕ А - Спецификация API
ПРИЛОЖЕНИЕ Б - Исходный код вспомогательных функций
ПРИЛОЖЕНИЕ 1 - Расширенные таблицы данных
```

**Примеры неправильного:**

```
ПРИЛОЖЕНИЕ А Спецификация             ❌ Нет дефиса
Приложение А - Спецификация           ❌ Не прописные буквы
ПРИЛОЖЕНИЕ А: Спецификация            ❌ Двоеточие вместо дефиса
```

### 3. ✓ Адресация из основного текста

Все приложения должны быть упомянуты в основном тексте документа.

**Поддерживаемые форматы ссылок:**

- `В приложении А` или `В Приложении А`
- `Из приложения Б`
- `Смотрите приложение В`
- `Appendix A`

**Пример:**

```
Подробное описание структуры базы данных
приведено в приложении А.

... различные варианты использования показаны
в приложении Б.

... полный исходный код находится в
приложении В.
```

### 4. ✓ Последовательность

При использовании букв: А, Б, В, Г, Д, Е, Ё, Ж, З, И...
При использовании цифр: 1, 2, 3, 4, 5...

## API

### Использование

```python
from app.services.validators.appendix_validator import AppendixValidator
from docx import Document

# Создаём валидатор
doc = Document('document.docx')
validator = AppendixValidator()

# Выполняем валидацию
result = validator.validate(doc, {})

# Результаты
if result.passed:
    print("✓ Все приложения оформлены правильно")
else:
    for issue in result.issues:
        print(f"• {issue.description}")
```

### Через ValidationEngine

```python
from app.services.validation_engine import ValidationEngine
from docx import Document

doc = Document('document.docx')
engine = ValidationEngine(profile='gost_7_32_2017')

results = engine.validate_document('document.docx')

# Проверяем результаты AppendixValidator
for result in results:
    if result['validator_name'] == 'AppendixValidator':
        if result['passed']:
            print("✓ Приложения в порядке")
        else:
            for issue in result['issues']:
                print(f"• {issue['description']}")
```

## Типичные ошибки

### ❌ Ошибка 1: Нарушена последовательность букв

```
ПРИЛОЖЕНИЕ А - ...
ПРИЛОЖЕНИЕ В - ...    ❌ Пропущено Б
```

**Исправление:**

```
ПРИЛОЖЕНИЕ А - ...
ПРИЛОЖЕНИЕ Б - ...
ПРИЛОЖЕНИЕ В - ...
```

### ❌ Ошибка 2: Смешанная нумерация

```
ПРИЛОЖЕНИЕ А - ...
ПРИЛОЖЕНИЕ 2 - ...    ❌ Буквы и цифры вместе
```

**Исправление:**
Используйте либо буквы, либо цифры:

```
ПРИЛОЖЕНИЕ А - ...
ПРИЛОЖЕНИЕ Б - ...
```

или

```
ПРИЛОЖЕНИЕ 1 - ...
ПРИЛОЖЕНИЕ 2 - ...
```

### ❌ Ошибка 3: Неправильный формат заголовка

```
ПРИЛОЖЕНИЕ:А Название             ❌ Двоеточие вместо пробела
Приложение А - Название           ❌ Не прописное
APPENDIX A - Title                 ❌ Английский вместо русского
```

**Исправление:**

```
ПРИЛОЖЕНИЕ А - Название           ✅
```

### ❌ Ошибка 4: Нет ссылок в основном тексте

```
[содержание основного текста без упоминания приложений]

ПРИЛОЖЕНИЕ А - Дополнительные данные   ❌ Приложение не упомянуто
```

**Исправление:**

```
Расширенные данные представлены в приложении А.

[далее основной текст]

ПРИЛОЖЕНИЕ А - Дополнительные данные   ✅
```

### ❌ Ошибка 5: Отсутствует название приложения

```
ПРИЛОЖЕНИЕ А -           ❌ Нет названия
```

**Исправление:**

```
ПРИЛОЖЕНИЕ А - Результаты экспериментов   ✅
```

## Рекомендации

✅ **Что делать**:

- Используйте либо буквы (А, Б, В...), либо цифры (1, 2, 3...), но не оба вместе
- Нумеруйте приложения последовательно, без пропусков
- Каждое приложение должно иметь название
- Ссылайтесь на приложения в основном тексте перед их размещением
- Печатайте "ПРИЛОЖЕНИЕ" ПРОПИСНЫМИ буквами
- Используйте дефис или тире между номером и названием

❌ **Чего избегать**:

- Нарушение последовательности нумерации
- Смешивание букв и цифр в нумерации
- Пропуски букв/цифр в нумерации
- Отсутствие ссылок на приложения в тексте
- Приложения с пустыми названиями
- Использование английского языка для слова "ПРИЛОЖЕНИЕ"

## Примеры полных документов

### Правильный документ с 3 приложениями

```
ВВЕДЕНИЕ

... описание структуры системы в приложении А.

ОСНОВНАЯ ЧАСТЬ

Подробная спецификация API представлена в приложении Б.

Исходный код находится в приложении В.

ЗАКЛЮЧЕНИЕ

...

ПРИЛОЖЕНИЕ А - Структура базовых компонентов

[содержание приложения]

ПРИЛОЖЕНИЕ Б - Спецификация API

[содержание приложения]

ПРИЛОЖЕНИЕ В - Исходный код вспомогательных функций

[содержание приложения]
```

### Неправильный документ

```
[основное содержание без упоминания приложений]

ПРИЛОЖЕНИЕ 1 - ...     ❌ На приложение не ссылаются
ПРИЛОЖЕНИЕ В - ...     ❌ Смешана нумерация (цифра и буква)
ПРИЛОЖЕНИЕ Г - ...     ❌ Пропущены А, Б, В
```

## Ограничения

1. **Только русский текст**: Валидатор в основном ищет русский стиль нумерации
2. **Простые паттерны**: Сложные форматирования (например, с цветом) могут быть пропущены
3. **Ссылки в основном тексте**: Если приложения упоминаются только в содержании, это может не обнаружиться

## Производительность

- **Скорость**: ~50-100ms на документ
- **Память**: ~1MB
- **Масштабируемость**: Поддерживает документы с 100+ приложениями
