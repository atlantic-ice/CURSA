# AdvancedFormatValidator Документация

## 📋 Описание

**AdvancedFormatValidator** проверяет расширенные правила форматирования документа согласно ГОСТ 7.32-2017.

Валидатор проверяет:

- ✅ Отступы абзацев
- ✅ Использование табуляции
- ✅ Мягкие переносы в тексте
- ✅ Оформление источников в библиографии
- ✅ Правильное оформление ссылок и сносок

## 🎯 Правила ГОСТ

### Правило 6: Отступы абзацев

**Требование:** Все абзацы должны иметь единообразный отступ первой строки (не более 3 см от левого края).

**Примеры допустимого:**

```
[отступ 1.25 см] Это абзац с правильным отступом.
[отступ 1.25 см] Все абзацы одинаковые в своём отступе.
```

**Примеры ошибок:**

```
[нет отступа] На этот абзац отступ не применен.
[отступ 5 см] Этот отступ слишком большой.
[отступ 0.5 см] Отступ неправильный размер.
```

### Правило 7: Табуляция

**Требование:** Документ должен использовать табуляцию только в таблицах. В основном тексте использование `Tab` недопустимо при форматировании абзацев.

**Примеры допустимого:**

```
| Таблица | Использование |        Табуляции | Разрешено |
| ------- | ------------- | ----------------| --------- |
```

**Примеры ошибок:**

```
	Это начинается с Tab вместо правильного отступа.
Параграф	с	несколькими	Tab	символами	неправильно.
```

### Правило 12: Гефтование (мягкие переносы)

**Требование:** Мягкие переносы (soft hyphens) в документе недопустимы. Переносы должны быть автоматическими или вообще отсутствовать.

**Примеры ошибок:**

```
Много­об­ра­зие ви­до­вой цел­но­сти...
```

### Правило 25: Оформление источников

**Требование:** Все элементы библиографического списка должны содержать:

- Фамилия Инициалы автора
- Полное название работы
- Год издания
- Место издания
- Издательство

**Примеры допустимого:**

```
[1] Иванов И.И. Наука о животных. 2020. М.: Наука.
[2] Петров П.П., Сидоров С.С. Основы методологии. 2019. СПб.: СПбГУ.
```

**Примеры ошибок:**

```
[1] Иванов И.И. Наука о животных.  # Отсутствует год
[2] Петров П.П. Основы. 2019.  # Отсутствует место издания
[3] "Какая-то книга". Издание.  # Плохое форматирование
```

### Правило 26: Оформление ссылок и сносок

**Требование:** Все ссылки должны быть оформлены скобками `[1]` и должны соответствовать номерам источников в списке литературы.

**Примеры допустимого:**

```
Согласно исследованиям [1], это явление хорошо изучено.
В работах [2], [3] показаны различные подходы.
```

**Примеры ошибок:**

```
Согласно исследованиям1, это явление без скобок.
В работах [99] ссылка на несуществующий источник.
Согласно [1], [1], [1] - повторяющиеся ссылки.
```

```python

# AdvancedFormatValidator включён по умолчанию
```

### Отдельно

```python
from app.services.validators import AdvancedFormatValidator

doc = Document('document.docx')

```

## 🐛 Типы ошибок

### Правило 6: Отступы

| Тип                    | Уровень | Описание                      |
| ---------------------- | ------- | ----------------------------- |
| `indent_missing`       | ERROR   | Абзац без отступа             |
| `indent_too_large`     | WARNING | Отступ > 3см                  |
| `indent_inconsistency` | WARNING | Отступ отличается от соседних |
| `indent_wrong_value`   | WARNING | Неправильное значение отступа |

### Правило 7: Табуляция

| Тип                     | Уровень | Описание                    |
| ----------------------- | ------- | --------------------------- |
| `tab_usage`             | ERROR   | Табуляция в основном тексте |
| `tab_instead_of_indent` | ERROR   | Tab вместо отступа          |

### Правило 12: Переносы

| Тип                | Уровень | Описание                   |
| ------------------ | ------- | -------------------------- |
| `soft_hyphen`      | WARNING | Обнаружены мягкие переносы |
| `multiple_hyphens` | WARNING | Несколько подряд переносов |

### Правило 25: Источники

| Тип                 | Уровень | Описание                  |
| ------------------- | ------- | ------------------------- |
| `missing_year`      | ERROR   | Источник без года         |
| `missing_publisher` | ERROR   | Отсутствует издательство  |
| `missing_place`     | ERROR   | Отсутствует место издания |
| `invalid_format`    | WARNING | Неправильный формат       |

### Правило 26: Ссылки

| Тип                   | Уровень | Описание                          |
| --------------------- | ------- | --------------------------------- |
| `invalid_reference`   | ERROR   | Ссылка на несуществующий источник |
| `missing_brackets`    | WARNING | Ссылка без скобок                 |
| `duplicate_reference` | WARNING | Повторяющиеся ссылки              |

## ✅ Рекомендации по исправлению

### Проблема: Неправильные отступы

**Диагноз:**

```
Абзац имеет отступ 1.5см, остальные 1.25см
```

**Решение:**

1. Выделить проблемный абзац
2. Помощь → Абзац... (Format → Paragraph)
3. На вкладке "Отступы и интервалы" установить:
   - Специальный отступ: Первая строка - 1.25см
4. Применить ко всем остальным

### Проблема: Tab в основном тексте

**Диагноз:**

```
Найдена табуляция в абзаце вместо отступа
```

**Решение:**

1. Найти параграф со словом (Ctrl+H)
2. Найти: `^t` (Tab символ)
3. Заменить на: пусто или пробел
4. ИЛИ переделать абзац отступом вместо Tab

### Проблема: Мягкие переносы

**Диагноз:**

```
Обнаружены мягкие переносы в слове 'научно-исследовательский'
```

**Решение:**

1. Найти и заменить (Ctrl+H)
2. Найти: `^-` (мягкий перенос)
3. Заменить на: пусто
4. Позволить Word переносить автоматически

### Проблема: Неполный источник

**Диагноз:**

```
Источник [3] не содержит года издания
```

**Решение:**

1. Перейти к источнику [3] в списке литературы
2. Добавить год: 2020
3. Пример правильного формата:
   ```
   [3] Автор А.А. Название работы. 2020. М.: Издательство.
   ```

### Проблема: Неправильная ссылка

**Диагноз:**

```
Ссылка [99] указывает на несуществующий источник
```

**Решение:**

1. Найти в тексте ссылку [99]
2. Если ошибка нумерации - перенумеровать
3. Если неправильный номер - изменить на [1], [2] и т.д.
4. Убедиться, что источник есть в списке литературы

## 🔍 Алгоритм работы

### 1. Проверка отступов (Правило 6)

```
Для каждого параграфа:
  1. Получить значение отступа первой строки
  2. Если отступ != 1.25см:
     - WARNING если не кратен или > 3см
  3. Если расхождение с соседними:
     - WARNING о неправильности
```

### 2. Проверка табуляции (Правило 7)

```
Для каждого параграфа вне таблиц:
  1. Проверить на наличие символа \t (Tab)
  2. Если найден в начале:
     - ERROR: используется вместо отступа
```

### 3. Проверка мягких переносов (Правило 12)

```
Для каждого параграфа:
  1. Найти мягкие переносы (U+00AD)
  2. Если найдены:
     - WARNING с позицией
```

### 4. Проверка источников (Правило 25)

```
Для каждого источника в библиографии:
  1. Проверить на год издания
  2. Проверить на место издания
  3. Проверить на издательство
  4. Если что-то отсутствует:
     - ERROR о пропуске
```

### 5. Проверка ссылок (Правило 26)

```
Для каждой ссылки в документе:
  1. Извлечь номер из скобок [N]
  2. Проверить что источник [N] существует
  3. Если не существует:
     - ERROR о неправильной ссылке
```

## 💡 Примеры использования

### Пример 1: Проверка всех 5 правил

```python
from app.services.validators import AdvancedFormatValidator
from docx import Document

doc = Document('thesis.docx')
validator = AdvancedFormatValidator()
result = validator.validate(doc, {})

if result.passed:
    print("✅ Документ успешно прошел проверку")
else:
    print(f"❌ Найдено {len(result.issues)} проблем:")
    for issue in result.issues:
        print(f"  - Правило {issue.rule}: {issue.message}")
```

### Пример 2: Проверка конкретного правила

```python
# Если нужно проверить только отступы (правило 6)
issues = [i for i in result.issues if i.rule == 6]
if issues:
    print(f"Проблемы с отступами: {len(issues)}")
```

### Пример 3: Получение рекомендаций

```python
result = validator.validate(doc, {})

# Получить типичные ошибки
errors_by_type = {}
for issue in result.issues:
    if issue.type not in errors_by_type:
        errors_by_type[issue.type] = []
    errors_by_type[issue.type].append(issue)

# Вывести группированным образом
for issue_type, issues in errors_by_type.items():
    print(f"{issue_type}: {len(issues)} проблем")
```

## 📝 Примечания

1. **Отступы** - должны быть единообразными. Используйте "Параметры абзаца" вместо Tab или пробелов.

2. **Табуляция** - если вы видите Tab при поиске `^t`, это нужно заменить на правильный отступ.

3. **Переносы** - Word может добавлять мягкие переносы автоматически. Используйте "Поиск и замена" для удаления.

4. **Ссылки** - номера начинаются с [1] и должны непрерывно идти до [N], где N - количество источников.

5. **Источники** - должны быть полными. При добавлении ссылки убедитесь, что на слайде вся необходимая информация.

## 🎓 ГОСТ Справочники

- **ГОСТ 7.32-2017** - основной стандарт оформления
- **ГОСТ 7.80-2000** - библиографическое описание
- **ISO 690** - международный стандарт ссылок

✅ Используйте шаблоны ВУЗа если они есть
