# Программный комплекс для автоматизации нормоконтроля (CURSA)

## Обновления системы - Февраль 2026

### Реализованная функциональность

Разработана модульная система валидации документов с архитектурой расширяемых валидаторов.

---

## ✨ Новая архитектура валидации

### Модульная система валидаторов

#### Базовый класс `BaseValidator`
**Файл:** `backend/app/services/validators/__init__.py`

Предоставляет единый интерфейс для всех валидаторов:
- `validate()` - метод проверки документа
- `Severity` enum - уровни серьезности (CRITICAL, ERROR, WARNING, INFO)
- `ValidationIssue` - класс для представления проблем
- `ValidationResult` - результат валидации с метриками

Преимущества:
✅ Единый интерфейс для всех проверок
✅ Типизированные результаты
✅ Автоматическое логирование проблем
✅ Поддержка автокоррекции

#### Реализованные валидаторы

##### 1. FontValidator
**Файл:** `backend/app/services/validators/font_validator.py`

Проверяет форматирование шрифтов:
- ✅ Название шрифта (Times New Roman, Arial и т.д.)
- ✅ Размер шрифта (14pt для основного текста, 10-12pt для таблиц)
- ✅ Цвет шрифта (черный)
- ✅ Консистентность по всему документу
- ✅ Шрифты в таблицах

**Автокоррекция:** Да
**Производительность:** ~0.1-0.5с для документа 50 страниц

##### 2. MarginValidator
**Файл:** `backend/app/services/validators/margin_validator.py`

Проверяет поля страницы:
- ✅ Все 4 поля (левое 3см, правое 1.5см, верх/низ 2см)
- ✅ Поля колонтитулов
- ✅ Консистентность полей между секциями
- ✅ Допуск ±0.2см для учета погрешностей

**Автокоррекция:** Да
**Производительность:** ~0.05-0.1с

##### 3. BibliographyValidator
**Файл:** `backend/app/services/validators/bibliography_validator.py`

Проверяет список литературы:
- ✅ Наличие раздела "Список использованных источников"
- ✅ Минимальное количество источников (настраивается профилем)
- ✅ Формат ГОСТ Р 7.0.5-2008 (автор, интернет-источники, ГОСТы, законы)
- ✅ Актуальность источников (возраст не более N лет)
- ✅ Правильность библиографических ссылок в тексте [1], [5, с. 28]
- ✅ Проверка процитированности всех источников

**Автокоррекция:** Частично (нумерация)
**Производительность:** ~0.2-0.5с

### ValidationEngine - оркестратор

**Файл:** `backend/app/services/validation_engine.py`

Управляет всеми валидаторами и генерирует итоговый отчет:

**Функции:**
- Запуск всех валидаторов параллельно
- Агрегация результатов
- Подсчет статистики (критические/ошибки/предупреждения)
- Вычисление процента соответствия требованиям
- Генерация рекомендаций по исправлению
- Определение общего статуса документа

**Метрики отчета:**
- Общее количество проблем
- Разбивка по серьезности
- Время выполнения каждого валидатора
- Количество автоисправимых проблем
- Процент выполнения требований

## 🚀 Новые API Endpoints

### POST /api/validation/check
Полная валидация документа.

**Параметры:**
- `file` (multipart) - DOCX файл
- `profile_id` (optional) - ID профиля требований

**Ответ:**
```json
{
  "status": "failed",
  "passed": false,
  "summary": {
    "total_issues": 15,
    "critical": 1,
    "errors": 8,
    "warnings": 5,
    "info": 1,
    "autocorrectable": 10,
    "completion_percentage": 85.0
  },
  "validators": [...],
  "issues_by_severity": {...},
  "recommendations": [...]
}
```

### POST /api/validation/quick-check
Быстрая проверка (базовые метрики без глубокого анализа).

### GET /api/validation/validators
Список доступных валидаторов и их статус.

### GET /api/validation/profiles
Список доступных профилей требований.

**Документация:** `backend/app/services/validators/README.md`

        issues = []

        # Логика проверки
        # ...

        return ValidationResult(
            validator_name=self.name,
            passed=len(issues) == 0,

## 📋 Профили требований

Профили определяют требования различных университетов/стандартов.

**Расположение:** `backend/profiles/*.json`

**Доступные профили:**
- ✅ ГОСТ 7.32-2017 (стандарт для НИР)
- ✅ ГОСТ Р 7.0.100-2018
- ✅ МГУ, МФТИ, ИТМО, ВШЭ, СПбГУ
- ✅ РАНХиГС, МГТУ им. Баумана
- ✅ Финансовый университет
- ✅ И другие (13+ профилей)

**Структура профиля:**
```json
{
  "name": "Название",
  "version": "1.0",
  "rules": {
    "font": {...},
    "margins": {...},
    "bibliography": {...}
  },
  "validation": {
    "check_font": true,
    "check_margins": true
  }
}
```

Каждый профиль может:
- Определять свои требования
- Включать/отключать валидаторы
- Настраивать допуски и параметры

## 🚀 Использование

### Для разработчиков:

```python
from app.services.validation_engine import ValidationEngine

# Создание движка валидации
engine = ValidationEngine(profile={'name': 'ГОСТ 7.32-2017', ...})

# Валидация документа
report = engine.validate_document('document.docx')

# Результаты
print(f"Статус: {report['status']}")
print(f"Процент соответствия: {report['summary']['completion_percentage']}%")
print(f"Найдено проблем: {report['summary']['total_issues']}")

# Рекомендации
for rec in report['recommendations']:
    print(f"- {rec}")
```

### Для API:

```bash
# Валидация документа
curl -X POST http://localhost:5000/api/validation/check \
  -F "file=@document.docx" \
  -F "profile_id=gost_7_32_2017"

# Список валидаторов
curl http://localhost:5000/api/validation/validators

# Список профилей
curl http://localhost:5000/api/validation/profiles
```
